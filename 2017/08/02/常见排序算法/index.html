基本排序算法

面临面试，虽然身为前端也要了解一些常用的排序算法。

插入排序

平均时间复杂度为O(N^2)，但是在基本预先排序，那么插入排序会很快。

希尔排序

希尔排序使用一个增量序列，在使用增量为h的一趟排序之后，对于每一个i，我们都有a[i]<=a[i+h]。所有相隔元素h的元素都被排序，此时称文件是h排序的。

h为一个增量序列，如1、3、5。当h为1时即完成排序。

希尔本人建议的序列是最大的h为N/2，hk=hk+1/2

堆排序

主要思路是利用大顶堆。

  1. 将初始的待排序数组构建成大顶堆。
  2. 将该堆的第一个元素与最后一个元素交换。则最后一个元素为最大值
  3. 将前n-1个元素构建新的大顶堆。再将第一个元素与n-1个元素交换。不断重复。直到有序元素个数为n-1

==================================================

关于堆的一些前置知识：

堆（二叉堆）可以视为一棵完全的二叉树

对于给定的某个结点的下标 i，可以很容易的计算出这个结点的父结点、孩子结点的下标：

  • Parent(i) = floor(i/2)，i 的父节点下标
  • Left(i) = 2i，i 的左子节点下标
  • Right(i) = 2i + 1，i 的右子节点下标

最大堆：

  • 最大堆中的最大元素值出现在根结点（堆顶）
  • 堆中每个父节点的元素值都大于等于其孩子结点（如果存在）

最小堆：

  • 最小堆中的最小元素值出现在根结点（堆顶）
  • 堆中每个父节点的元素值都小于等于其孩子结点（如果存在）

===================================================

归并排序

时间复杂度O(NlogN)

这个算法的基本操作是合并两个已排序的表。基本的合并算法是取两个输入数组A和B，一个输出数组C，以及3个计数器Actr、Bctr、Cctr，它们的初始位置对应于数组的开始端。A[Actr]和B[Bctr]中的较小者被拷贝到C中的下一个位置，相关的计数器向前推进一步。当两个输入表有一个用完的时候，则将另个表中的剩余部分拷贝到C中。

快速排序

主要利用分治法，将快速排序分为三步：

  1. 在数据集之中选择一个元素作为“基准”
  2. 所有小于基准的元素都移到基准的左边，所有大于基准的元素，都移到基准的右边
  3. 对基准左边和右边的两个子集，不断的重复第一步和第二步，直到所有的子集只剩下一个元素为止

更简单的语法（ES6）

上面简单版本的缺点是，它需要O(n)的额外存储空间，也就跟归并排序一样不好。额外需要的存储空间配置，在实际上的实现也会极度影响速度和高速缓存的性能。
