第一章 块级作用域绑定

var声明及变量提升机制

在函数作用域或全局作用域中通过关键字var声明的变量，无论实际上是在哪里声明的，都会被当成在当前作用域顶部的声明。

ES5是只有函数作用域的，ES6引入块级作用域来强化对变量生命周期的控制。

块级声明

  1. let声明

    • let声明的用法与var相同。用let代替var来声明变量，就可以把变量的作用域限制在当前代码块中。
    • let声明不会被提升
    • 假设作用域中已经存在某个标识符，此时再使用let关键字声明它就会出错

  2. const声明

    • 使用const声明的是常量，其值一旦被设定后不可更改。
    • 每个通过const声明的常量必须进行初始化
    • const和let声明都是块级标识符，常量也只在当前代码块内有效，一旦执行到块外会立即被销毁
    • 无论是在严格模式还是非严格模式下，都不可以为const定义的常量再赋值。但是如果const常量的值为对象，对象中的值可以修改(因为指针不变)

  3. 临时死区

    • 与var不同，let和const声明的变量不会被提升到作用域顶部，如果在声明之前访问这些变了，即使是相对安全的typeof操作符也会触发引用错误
    • js引擎在扫描代码发现变量声明时，要么将它们提升至作用域顶部（遇到var声明），要么将声明放到TDZ(临时死区)中（let和const）。访问TDZ中的变量会触发运行时错误

  • 在循环中用let代替var。在for循环之后不能再访问

循环中的let声明

  • 使用let可以很好的解决上面问题
  • let声明在循环内部的行为是标准中专门定义的，它不一定与let的不提升特性相关

循环中的const声明

  • for循环中由于const变量的值不能被修改所以会出错
  • 在for-in和for-of循环中使用const时的行为与使用let一致

全局块作用域绑定

  • 当var被用于全局作用域时，它会创建一个新的全局变量作为全局对象的属性。这意味着用var很可能会无意中覆盖一个已经存在的全局变量
  • 如果你在全局作用域中使用let或const，会在全局作用域下创建一个新的绑定，但该绑定不会添加为全局对象的属性

块级绑定最佳实践

  • 默认使用const，只用确实需要改变变量的值时使用let。因为大部分变量的值在初始化后不应再改变

第二章 字符串和正则表达式

更好的Unicode支持

  1. UTF-16码位

    背景知识：UTF-16中，前2^16个码位均以16位的编码单元表示，这个范围为基本多文种平面（BMP），超出这个范围的码位则要归属于某个辅助平面。为此，UTF-16引入了代理对，规定用两个16位编码单元表示一个码位。

    • 如果有通过代理对表示的字符。那么字符的length属性为2，charAt属性不会返回合法字符串，charCodeAt()会返回每个16位的编码

  2. codePointAt()方法

    • ES6新增完全支持UTF-16的codePointAt()方法，这个方法接受编码单位的位置而非字符位置作为参数，返回与字符串中给定位置对应的码位

    • 检测一个字符占用的编码单元数量，可以写这样的函数：

  3. String.fromCodePoint()方法

    • codePointAt()方法的反向，根据指定的码位生成一个字符。
    • 更完整版的String.fromCharCode()

  4. normalize()方法

    • 它可以提供Unicode的标准化形式。这个方法接收一个可选的字符串参数，指明应用以下的某种Unicode标准化形式。（“NFC”、“NFD”、“NFKC”，“NFKD”）
    • 由于不同的字符进行排序或比较操作有可能存在它们是等效的可能，所以在比较字符串之前最好标准化为同一形式

  5. 正则表达式u修饰符

    • 当一个正则表达式添加了u修饰符之后，它就从编码单元操作模式切换位字符模式。如此以来正则就不会视代理对为两个字符了

    • 检测u修饰符

其他字符串变更

  1. 字符串中的子串识别
    • includes()：字符串中检测到指定文本则返回true，否则返回false
    • startsWith()：如果在字符串的起始部分检测到指定文本则返回true，否则返回false
    • endsWith()：和startsWith类似
    • 这3个方法都接受两个参数：第一个参数指定要搜索的文本，第二个参数是可选的，指定一个开始搜索的位置的索引值。
    • endsWith()的第二个参数从字符串长度减去这个索引值的位置开始匹配

模版字面量

为了让开发者解决更复杂的问题，ES6模版字面量语法支持创建领域专用语言(DSL)

ES6弥补了ES5中缺少的特性，多行字符串、基本的字符串格式、HTML转义

  1. 基础语法

    • 模版字面量最简单的用法，看起来好像只是用反撇号`，代替了单、双引号。
    • 如果想在字符串中使用反撇号，那么用反斜杠将它转义就可以

  2. 多行字符串

    • ES5想实现多行字符串都是依靠数组或者字符串拼接的方法
    • ES6中只要直接换行即可。注意使用模版字面量时所有的空白符都属于字符串的一部分，所以要小心缩进。
    • 需要通过缩进来对齐文本，可以第一行留白，然后使用trim()方法

  3. 字符串占位符

    • 在模版字面量中，你可以把任何合法的JavaScript表达式嵌入到占位符中并将其作为字符串一部分输出
    • 占位符：${}，中间包含任意js表达式
    • 他的名字是${person.name}

  4. 标签模版

    • 标签指的是在模版字面量第一个反撇号(`)前方标注的字符串。

    • 标签可以是一个函数，调用时传入加工过的模版字面量各部分数据，但必须结合每个部分来创建结果

    • 这个函数的第一个参数是一个数组，包含JS解释过程后的字面量字符串，之后的所有参数都是每一个占位符的解释值

    • 标签模版的重要应用，就是过滤HTML内容

    • String.raw转义斜杠

明确函数的多重用途

JS函数有两个不同的内部方法：[[Call]]和[[Constructor]]，当使用new关键字调用函数时，执行[[Constructor]]方法，它负责创建一个通常被称为实例的新对象，然后再执行函数体，将this绑定到实例上；如果不通过new，则执行[[Call]]函数，直接执行函数体中代码

  1. ES5中判断函数被调用的方法（确定函数是否通过new关键字被调用）
    • 一般通过判断this instanceof Person
    • 如果使用call方法(var nperson = Person.call(person,"a"))，以上方法失效
  2. 元属性(Metaproperty)new.target
    • 为了判断函数是否通过new关键字调用的问题，ES6引入了new.target这个元属性
    • 当调用函数的[[Construct]]方法时，new.target（在构造函数中）被赋予为new操作符的目标，通常是新创建对象的实例。如果调用[[Call]]方法，则new.target的值为undefined

箭头函数

  1. 差异

    • 没有this、super、arguments和new.target绑定 箭头函数中的this、super、arguments及new.target这些值由外层最近一层非箭头函数决定
    • 不能通过new关键字调用 箭头函数没有[[Construct]]方法，所以不能用作构造函数，如果通过new关键字调用，程序会抛出错误。
    • 没有原型 由于不可以通过new关键字调用箭头函数，因而没有构建原型的需求，所以箭头函数不存在prototype这个属性
    • 不可以改变this的绑定 函数内部的this值不可以被改变，在函数的生命周期内始终一致。
    • 不支持arguments对象 箭头函数没有arguments绑定，所以你必须通过命名参数和不定参数这两种形式访问函数的参数
    • 不支持重复的命名参数 无论在严格还是非严格模式下，箭头函数都不支持重复的命名参数；而在传统函数的规定中，只有在严格模式下才不能有重复的命名参数

第四章 扩展对象的功能性

对象类别

  • 普通(Ordinary)对象：具有JavaScript对象所有的默认内部行为。
  • 特异(Exotic)对象：具有某些与默认行为不符的内部行为。
  • 标准(Standard)对象：ECMAScript6规范中定义的对象，例如：Array、Date等。标准对象既可以是普通对象，也可以是特异对象。
  • 内建对象：脚本开始执行时存在于JavaScript执行环境中的对象，所有标准对象都是内建对象

新增方法

  1. Object.is()方法

    • ES6引入Object.is()方法来弥补全等运算符的不准确运算。这个方法接受两个参数，如果这两个参数类型相同且具有相同的值，则返回true

  2. Object.assign()方法

    • 混合(Mixin)是JavaScript中实现对象组合最流行的一种模式。在一个mixin方法中，一个对象接收来自另一个对象的属性和方法

    • Object.assign()方法实现混合模式，这个方法接受一个接收对象和任意数量的源对象，最终返回接收对象。这个接收对象具有源对象们的属性和方法

    • 如果多个源对象具有同名属性，则排位靠后的源对象会覆盖排位靠前的

    • Object.assign()方法不能将提供者的访问器属性复制到接收对象中，由于Object.assign()方法执行了赋值操作，因此提供者的访问器属性最终会转变为接受对象的一个数据属性

重复的对象字面量属性

ES5中严格模式加入了对象字面量重复属性的校验。有重复属性报错

ES6中对每组重复属性，都会选取最后一个取值。

自由属性枚举顺序

  • ES5中未定义对象属性的枚举顺序，而ES6中严格规定了对象的自有属性被枚举时的返回顺序。这会影响到Object.getOwnPropertyName()方法及Reflect.ownKeys
  • 顺序的基本规则：
    1. 所有的数字键按升序排序
    2. 所有的字符串键按照它们被加入对象的顺序排序
    3. 所有symbol键按照它们被加入对象的顺序排序
  • 对于for-in循环，Object.keys()和JSON.stringify()方法的枚举顺序目前不明晰

增强对象的原型

  1. 改变对象的原型

    • Object.setPrototypeOf()方法，通过这个方法可以改变任意指定对象的原型
    • 正常情况下，无论是通过构造函数还是Object.create()方法创建对象，其原型是在对象被创建时指定的。对象原型在实例化后保持不变
    • 对象原型的真实值被存储在内部专用属性[[Prototype]]中，调用Object.getPrototypeOf()方法返回储存在其中的值，调用Object.setPrototypeOf()方法改变其中的值

  2. 简化原型访问的Super引用

    • 如果你想重写对象实例的方法，又需要调用与它同名的原型方法：

    • ES6引入super关键字，简单来说，Super引用相当于指向对象原型的指针，实际上也就是Object.getPrototypeOf(this)的值

    • 必须要在简写方法的对象中使用Super引用，不然会报错

    • super在多重继承的情况下很有用，因为Object.getPrototypeOf(this)可能形成递归。而Super引用不是动态变化的。

  • ES6中正式将方法定义为一个函数，它会有一个内部的[[HomeObject]]属性来容纳这个方法从属的对象。
