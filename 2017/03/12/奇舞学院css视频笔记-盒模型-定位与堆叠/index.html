CSS第四课

盒模型

width

  • 指定content box宽度
  • 百分数相对于父容器(包含块)的content box宽度

height

  • 指定content box宽度
  • 百分数相对于父容器(包含块)的content box宽度
  • 只有当包含块的高度不依赖该元素时，百分比高度才生效

padding

  • 内边距
  • padding-top、padding-right、padding-bottom、padding-left
  • 缩写：padding

margin

  • 外边距
  • margin-top、margin-right、margin-bottom、margin-left
  • 缩写：margin

上下margin会折叠

margin可以为负值

box-sizing(CSS3)

  • 改变盒模型计算方式
  • 取值：border-box | content-box
  • 初始值：content-box

给html加上border-box然后再在需要content-box的地方使用

border

  • 边框的三个要素
    • border-width
    • border-style：none | solid | dashed | dotted | double
    • border-color
  • 四个方向
    • border-left
    • border-top
    • border-right
    • border-bottom

可以通过边框来做一个三角形，即将三条边框透明，中间content为0

min-width&max-width

最小和最大宽度

高度同理

overflow

  • 溢出控制
  • 取值：visible | hidden | scroll | auto

视觉格式化模型

浏览器是怎么排版的？

视口（Viewport）

  • 浏览器的可视区域
  • 用户通过视口查看网页内容

块级元素

  • 会呗格式化成块状的元素
  • 例如p、div、section等
  • 将display设置为block、list-item、table使元素变为块级

行级元素

  • 不会为其内容生成块级框
  • 让其内容分布在多行中
  • display设置为inline、inline-block、inline-table使元素变为行级

盒子的生成

  • 每个块级元素生成一个主块级盒(principal block-level box)，用它来包含子级盒子
  • 每个行级元素生成一个行级盒，行级盒分布于多行

Box Model - revisited

  • margin：行级盒的margin-top和margin-bottom不会产生效果（左和右会有效果）
  • padding：行级盒的padding-top和padding-bottom不影响行布局

块级盒子中的子盒子的生成

  • 块级盒子中可以包含多个子块级盒子
  • 也可以包含多个行级盒子
  • 不在行级元素里面的文字，会生成匿名行级盒。比如<p>Some <em>Text</em></p>some就生成匿名盒子
  • 块级盒子中不能同时包含块级和行级盒子。遇到这种情况时，会生成匿名块级盒来包含行级盒。比如<div><h1>h1</h1><span>2016-1-1</span></div>

行级盒子内的子盒子的生成

  • 行级盒子内可以包含行级盒子
  • 行级盒子包含一个块级盒子时，会被块级盒子拆成两个行级盒子，这两个盒子又分别被匿名块级盒子包裹

原则：块级和行级盒子不能并列

display属性

  • block生成块级盒
  • inline生成行级盒
  • inline-block生成行级盒，为其内容生成块级盒
  • none在排版时将元素忽略

visibility

  • 控制元素展示
  • 取值：visible | hidden | collapse
  • 初始值为visible

Generated Content

  • 控制元素
  • ::before和::after
  • content

通过css的伪元素生成盒子

CSS第五课

定位与堆叠

定位模式

  • 常规流(Normal Flow)
  • 浮动(Float)
  • 绝对定位

position

  • static，非定位，默认值
  • relative，相对定位(相对自己)
  • absolute，绝对定位，相对非static祖先元素定位
  • fixed，相对于视口绝对定位

relative

  • 在常规流里面布局
  • 相对于自己本应该在的位置进行偏移
  • 使用top、left、bottom、right设置偏移长度
  • 流内其他元素当它没有偏移一样布局

absolute

  • 脱离正常流
  • 相对于最近的非static祖先的padding box定位
  • 不会对流内元素布局造成影响
  • 可以有margin，但不会折叠

自动计算

  • 这些值都可以不指定

    • top

    • left

    • right

    • bottom

    • width

    • height

      auto是在该元素本该在的地方，只不过是脱离了流

  • 如果冲突了怎么办？

fixed

fixed相对于屏幕定位

高度和屏幕高度一样可以用vh单位

z-index堆叠层次

  • 为定位元素制定其在z轴的上下等级
  • 用一个整数表示，数值越大，越靠近用户
  • 初始值为auto，可以为负数、0、正数

line height和容器高度一样，文字垂直居中

z-index大的不一定会在上面，需要看上下文。

堆叠上下文(stacking Context)

堆叠上下文的生成

  • root元素
  • z-index值不为auto的定位元素
  • 设置了某些CSS3属性的元素，比如opacity、transform、animation等

绘制顺序

  • 在每一个堆叠上下文中，从下到上
    • 形成该上下文的元素的border和background
    • z-index为负值的子堆叠上下文
    • 常规流内的块级元素非浮动子元素
    • 非定位的浮动元素
    • 常规流内非定位行级元素
    • z-index为0的子元素或子堆叠上下文
    • z-index为正数的子堆叠上下文
