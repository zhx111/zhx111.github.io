提交更新

当你add修改文件之后，需要commit提交：

commit会将你add的所有文件提交到仓库中，-m后的命令为这次提交的注释，虽然不是必须的，但最好一定要写，这样对别人或者是以后看代码的自己都有很大的好处。

如果你觉得一个一个add后在提交太麻烦的话，可以使用$ git commit -a -m "xxx"，-a选项会让git把所有跟踪过的文件都提交。

远程仓库

添加

$ git remote命令可以查看当前配置的远程仓库，通过-v,可以显示对应的克隆地址。

可以添加一个新的远程仓库。

抓取数据

可以使用 git pull 命令自动抓取数据下来，然后将远端分支自动合并到本地仓库中当前分支，实际上，默认情况下 git clone 命令本质上就是自动创建了本地的 master 分支用于跟踪远程仓库中的 master 分支。

推送数据

该命令将当前master分支推送到了远端。加上了-u参数，Git不但会把本地的master分支内容推送给远程新的master分支，还会把本地的master分支和远程的master分支关联起来，在以后的推送或者拉取时就可以简化命令。

注意，只有具有ssh权限的账户才可以推送上去。
