JavaScript高程阅读笔记

第3章 基本概念

  1. 区分大小写
  2. 用var操作符定义的变量将成为定义该变量的作用域中的局部变量
  3. 数据类型
    • 五种简单数据类型：Undefined、Null、Boolean、Number和String。还有一种复杂数据类型——Object
    • typeof是一个操作符，用来检测给定变量的数据类型。typeof null返回object
    • undefined在使用var声明但是未对其加以初始化，null表示一个空指针对象，如果在一个变量准备在将来用于保存对象
    • Boolean的转化规则。
      • 数据类型 转换为true的值 转换为false的值
      • String 任何非空字符串 “”空字符串
      • Number 任何非零数字值 0和NaN（0会转换为false！）
      • Object 任何对象 null
      • Undefined n/a undefined
    • Number中，八进制字面值的第一位必须是零，然后是八进制数字序列(0-7)；十六进制字面值的前两位必须是0x，后跟任何十六进制数字
    • e表示法：var floatNum = 3.125e7;var floatNum2 =3e-17
    • 浮点数值的最高精度是17位小数，但在进行算术计算的时精度远远不如整数。比如0.1+0.2结果不是0.3而是0.300000…004。
    • Number.MIN_VALUE和Number.MAX_VALUE是ECMAScript的限制
    • NaN非数值，表示一个本来要返回数值的操作数未返回数值的情况。NaN和任何值都不相等。
    • ECMAScript提供isNaN()函数,判断是否参数是否是数值。
    • 三个数值转换函数：Number()、parseInt()和parseFloat()；第一个可以用于任何数据类型，后两个则专门把字符串转换成数值。
    • Number()函数转换规则:
      • 如果是Boolean，true和false分别转换为1和0
      • 如果是数字值，只是简单的传入和返回
      • 如果是null值，返回0
      • 如果是undefined，返回NaN
      • 如果是字符串，遵循以下规则：只包含数字，则转换为十进制数值；包含有效浮点格式，则转换为浮点数；包含有效的十六进制，则将其转换为相同大小的十进制整数值，字符串是空的则将其转换为0，字符串包含其他则转换为NaN
      • 如果是对象，则对象的valueOf()方法，然后依照前面的规则转换返回的值,如果结果是NaN则调用toString()
    • parseInt()会忽略字符串前面的空格，直到找到第一个非空格字符。如果第一个字符不是数字字符或者负号，返回NaN。如果后面有非数字字符就忽略
    • parseInt()第二个参数为进制数
    • parseFloat只有第一个小数点是有效的
    • ECMAScript中的字符串是不可变的
    • toString()方法转变为字符串，对于数值转字符串，可以接受一个参数，根据参数输入二进制、八进制、十六进制
    • Object的每个实例都具有下列属性和方法：
      • constructor
      • hasOwnProperty(propertyName):用于检查给定的属性在当前对象实例中是否存在（而不是在实例的原型中）
      • isPrototypeOf(object):用于检查传入的对象是否是当前对象的原型
      • propertyIsEnumerable(propertyName)：用于检查给定的属性是否能够用for-in语句
      • toString()、valueOf()
  4. 操作符
    • 位操作符
    • ECMAScript中的所有数值都以IEEE-754 64位格式存储但位操作符先将64位的值转换为32位的整数然后执行操作，最后再将结果转换为64位。
    • 对于有符号整数，32位中的前31位勇于表示整数，第32位表示符号。0正1负。负数已二进制补码存储
    • 按位非（NOT）~ 本质：操作数的负值减一
    • 按位与（AND）&
    • 按位或（OR）|
    • 按位异或(XOR) ^
    • 左移 << 不会影响操作数的符号位
    • 无符号右移 >>>
    • 同时使用!!模拟Boolean()转型函数的行为，将一个值转换为布尔值
    • 相等和不相等先比较再转换，全等和不全等，仅比较不转换
    • for…in语句，可以用来枚举对象的属性
    • label语句可以用于多循环嵌套的退出
    • js可以在switch语句中使用任何数据类型
    • switch语句在比较值时使用的是全等操作符
  5. 函数
    1. 理解参数
      • ECMAScript不介意传递进来多少参数，也不在乎传进来参数是什么数据类型
      • arguments对象接收函数的参数，arguments对象只是和Array类似，并不是Array的实例
    2. 没有重载 如果有两个名字相同的函数，则该名字只属于后定义的函数

第4章 变量、作用域和内存问题

  1. 基本类型和引用类型
    • 基本类型按值访问，引用类型不能操作对象的内存空间，只能操作对象的引用
    • 对于引用类型的值我们可以为其添加属性和方法，也可以改变和删除属性和方法；对于基本类型我们不能添加属性
    • ECMAScript中字符串为基本类型而不是引用类型
    • 如果从一个变量向另一个变量复制基本类型的值，会在变量对象上创建一个新值，然后把该值复制到为新变量分配的位置上；引用类型则是复制指针
    • ECMAScript中所有函数都是按值传递的，而引用类型的传递，则如同引用类型变量的复制，可以把该参数想象成局部变量
    • typeof检测基本类型，instanceof检测引用类型（根据原型链）
  2. 执行环境和作用域
    • 执行环境 定义了变量或函数有权访问的其他数据。每个执行环境都有一个与之关联的变量对象
    • 每个函数都有自己的执行环境，当代码在一个环境中执行时，会创建变量对象的一个作用域链，作用域链前端始终都是当前执行的代码所在环境的变量对象，即arguments对象。全局执行环境的变量对象始终是作用域链中的最后一个对象
    • 没有块级作用域（let解决了这个）
  3. 垃圾收集
    • 两个策略：标记清除、引用计数

第6章 面向对象的程序设计

理解对象

  1. 属性类型

    特性(attribute)描述了属性的各种特征。ECMA中有两种属性：数据属性和访问器属性。

    1. 数据属性

      • [[Configurable]]：表示能否通过delete删除属性从而重新定义属性，能否修改属性的特性，或者能否把属性修改为访问器属性

      • [[Enumerable]]：表示能否通过for-in循环返回属性。

      • [[Writable]]：表示能否修改属性的值。

      • [[Value]]:包含这个属性的数据值。读取属性的时候，从这个位置读；写入属性值的时候，把新值保存在这个位置。

      • 要修改属性默认的特性，必须使用ECMAScript5的Object.defineProperty()方法。这个方法接收三个参数：属性所在对象，属性名字和一个描述符对象。

    2. 访问器属性

      • 访问器属性不包含数据值；它们包含一对getter和setter函数
      • [[Configurable]]：表示能否通过delete删除属性从而重新定义属性，能否修改属性的特性，或者能否把属性修改为数据属性。
      • [[Enumerbale]]：同数据属性
      • [[Get]]：在读取属性时调用的函数。默认值为undefined
      • [[Set]]：在写入属性时调用的函数。默认值为undefined
      • 通过定义set和get函数可以实现数据和视图的双向绑定。（set新值时在函数里改变视图的值）

  2. 定义多个属性

    • Object.defineProperties()

  3. 读取属性的特性

    • 使用Object.getOwnPropertyDescriptor()方法，可以取得给定属性的描述符。这个方法接收两个参数：属性所在的对象和要读取其妙舒服的属性名称。返回值是一个对象，如果是个访问器属性，这个对象的属性有访问器的那四个特性。是数据属性，则是数据属性的那四个特性。

  3. 原型模式

    对于之前的缺点。可以用原型模型解决。

    每个函数都有一个prototype(原型)属性，这个属性是一个指针，指向一个对象，而这个对象的用途是包含可以由特定类型的所有实例共享的属性和方法。

    1. 理解原型对象

      当我们创建一个新函数（function Person(){}）的时候，会根据一组特定的规则为该函数创建一个prototype属性，这个属性指向函数的原型对象。

      在默认情况下，所有原型对象都会获得一个constructor属性，这个属性指向prototype属性所在的函数(Person())。通过这个构造函数，我们可以继续为原型对象添加其他属性和方法。

      创建了自定义构造函数（Person）之后，其原型对象只会取得constructor属性；其他方法则都从Object继承而来。

      当我们调用构造函数(Person)创建新实例(person1)时，该实例内部包含一个指针，指向构造函数的原型对象。这个指针ECMA称为[[Prototype]]实际浏览器中都支持__proto__属性取得。（还有Object.getPrototypeOf()）

      每当代码读取某个对象的某个属性的时，都会执行一次搜索，如果对象实例本身找到了具有给定名字的属性，则返回该属性的值。如果没找到，则继续搜索指针指向的原型对象，在原型对象中查找具有给定名字的属性。如果在原型对象中找到了这个属性，则返回该属性的值。

      虽然可以通过对象实例访问保存在原型中的值，但却不能通过对象实例重写原型中的值。如果我们在实例中添加了一个属性，而该属性与实例原型中的一个属性同名，那么我们就在实例中创建该属性，该属性会屏蔽原型中的那个属性。

      使用hasOwnProperty()方法可以检测一个属性是存在于实例中还是存在于原型中。

    2. 原型和in操作符

      有两种方式使用in操作符，单独使用和在for-in中使用。

      • 在单独使用时，in操作符会在通过对象能够访问给定属性时返回true，无论该属性存在于实例还是原型中
      • 在使用for-in循环时，返回的是所有能够通过对象访问的、可枚举的属性，其中既包含存在实例中，也包含原型中的属性。
      • 要取得对象上所有可枚举的实例属性，可以使用Object.keys()方法。该方法接收一个对象作为参数，返回一个包含所有可枚举属性的字符串数组。
      • 想得到所有实例属性，可是使用Object.getOwnPropertyNames()方法

继承（关键！）

  1. 原型链

    js实现继承主要是靠原型链实现的。其基本思想是利用原型让一个引用类型继承另一个引用类型的属性和方法。

    如果我们让一个构造函数的原型对象等于另一个类型的实例，由于另一类型的实例包含了指向该类型原型对象的指针__proto__，那么这个构造函数的实例所指向的原型对象也就包含了指向其父类型（上文中的另一类型）的原型

    sub继承super是通过创建Super的实例并通过将该实例赋给Sub.prototype实现的，这就是最简单的继承实现方式。

    此时instance.constructor指向的是Super

    1. 别忘记默认的原型

      • 所用引用类型继承了Object所以原型链顶端是Object的原型

    2. 确定原型和实例方法

      • 用instanceof和isPrototypeOf()都能确定原型和实例的关系。上例的instance是Sub、Super和Object的实例。

    3. 谨慎地定义方法

      • 子类型有时候需要覆盖超类型中的某个方法，或者需要添加超类型中不存在的某个方法。但不管怎么样，给原型添加方法的代码一定要放在替换原型语句之后。

      • 在通过原型链实现继承时，不能使用对象字面量创建原型方法，因为这样做会重写原型链

    4. 原型链的问题

      • 最主要的问题来自包含引用类型值的原型。因为引用类型值的原型会被所有实例所共享

        • 原型链的第二个问题是：在创建子类实例时，不能向超类型的构造函数中传递参数

  2. 借用构造函数

    实际上是通过用call方法，在子类构造函数内部调用超类的构造函数。通过call将Super作用域改为this（Sub的作用域），所以Sub的this上带有Super的属性副本。

    可以通过给call方法传递参数给超类传值

    • 问题

      如果仅仅借用构造函数，那么也无法避免构造函数模式存在的问题——方法都在构造函数中定义，则函数复用无可谈起。

  3. 组合继承

    针对以上的问题，给出组合继承。实现思路是通过借用构造函数实现对原型属性和方法的继承，通过原型链实现对原型属性和方法的继承

  4. 原型式继承

    借助原型基于已有的对象创建新对象，本质上讲，object()对传入的对象执行了一次浅复制

    ECMAScript 5通过新增Objcet.create()方法规范化了原型式继承。这个方法接收两个参数，一个用作新对象原型的对象，一个作为新对象定义额外属性的对象（可选）

第21章 Ajax与Comet

21.1 XMLHttpRequest

  • IE7+以及现代浏览器都支持XMLHttpRequest对象，var xhr = new XMLHttpReuqest();
  • open方法开启请求（三个参数发送类型、url和是否异步）,send方法作为请求主体发送数据。如果不需要通过请求主题发送数据，则必须传入null
  • 通过readystatechange事件来检测每次状态变化后的readyState值，当值为4的时候即位完成。同时，status属性保存着响应的http的状态，而response属性响应服务器返回的数据。
  • 收到响应之前可以调用abort()方法来取消异步请求
  • setRequestHeader()设置发出的http请求头，调用xhr对象的getResponseHeader()并传入头部字段名称取得相应的响应头部信息，调用getAllResponseHeaders()方法去的一个包含所有头部信息的长字符串
  • 使用get请求需要对查询字符串进行编码，使用encodeURIComponent()函数

21.2 XMLHttpRequest2级

  • FormData类型 为序列化表单以及创建与表单格式相同的数据（用于通过XHR传输）提供了便利。可以通过append方法向formdata中加入数据（键和值），也可以通过向FormData构造函数中传入表单元素
  • 超时设定 IE8+支持 给xhr对象添加了一个timeout属性，再给timeout设置一个数值后，如果在规定的时间内浏览器还没有接收到响应，就会触发timeout事件。
  • overrideMimeType()方法

21.3 进度事件

  • progress事件

    这个事件会在浏览器接收新数据期间周期性地触发。而onprogress事件处理程序会接收到一个event对象，其target属性是XHR对象，但包含三个额外的属性：lengthComputable、position和totalSize。可以通过这三个信息创建一个进度指示器。

21.4 跨域资源共享

  • 域：只要协议、域名、端口有任何一个不同，都被当作是不同的域。
  • CORS背后的基本思想，是是用自定义的HTTP头部让浏览器与服务器进行沟通，从而决定请求或响应是应该成功还是应该失败。
  • 客户端发送Origin头部包含请求页面的源信息，而服务器端发送Access-Control-Allow-Origin头部中回发相同的源信息。
  • 请求和响应都不包含cookie信息
  • 限制：不能使用setRequestHeader()设置自定义头部。不能发送和接收cookie。调用getAllResponseHeaders()方法总会返回空字符串。

21.5 其他跨域技术

  • 图像Ping

    利用了图像的src属性可以从任何网页中加载图像。最常用于跟踪用户点击页面或动态广告曝光次数。图像Ping有俩个主要的缺点，一是只能发送GET请求，二是无法访问服务器的响应文本。

  • JSONP（JSON with padding）

    JSONP由两部分组成：回调函数和数据。主要通过动态script使用。

  • Comet 长轮询和流实现

  • 服务器发送事件SSE（HTML5）

  • 创建EventSource对象

    • open在建立连接时触发，message在从服务器接收到新事件时触发，error在无法建立连接时触发
    • IE不支持

  • Web Sockets

    目标是在一个单独的持久连接上提供全双工、双向通信。

    • 创建WebSocket对象
    • send()发送数据，服务端返回数据触发message事件
