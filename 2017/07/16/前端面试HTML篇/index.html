语义

  1. 节段和提纲

    • HTML5增加了新的节段和提纲元素，比起以前单纯的使用div+class划分网页结构增强了语义性，这些元素分别是：

    <section><article><nav><header><footer><aside><hgroup>

    • <section>元素表示文档的一个区域或节，一般包含一个标题；<article>元素表示文档、页面、应用或网站中的独立结构，其意在成为可独立分配的或可复用的结构，
    • 其中，<hgroup>已经从HTML5（W3C）规范中删除，但是它仍旧在 WHATWG 的 HTML 版本里。

  2. 使用HTML5的音频和视频

    • audio音频元素

      <audio> 元素用于在文档中表示音频内容。 <audio> 元素可以包含多个音频资源， 这些音频资源可以使用 src 属性或者<source>元素来进行描述；

      属性：

      • autoplay 布尔属性，是否自动播放
      • buffed 你可以通过该属性获取已缓冲的资源的时间段信息
      • controls 该属性提供播放的控制面板
      • loop 布尔属性，是否循环播放
      • playload 枚举属性，none，metadata，auto。浏览器使用何种方式加载
      • 更多属性参考：MDN

      事件：

      • 可以通过js操作audio上的事件，具体参考：MDN:media-event

    • video视频元素

      • video元素大体和audio元素差不多，不过它的属性多了一个crossorigin（该枚举属性指明抓取相关图片是否必须用到CORS（跨域资源共享）），poster（播放之前的海报帧），height和width
      • 如果视频的MIME属性设置不正确，视频可能会不显示。
      • 通过canvas 处理video可以看这篇：canvas

  3. 新的语义元素

    • <mark>高亮语句、表现上下文之间的关联性。比如搜索结果
    • <figure>代表一段独立的内容，说明(caption) <figur> 配合使用
    • <output>
    • <progress>显示一项任务的完成情况，属性有max和value
    • <meter>显示已知范围的标量值或者分数值

    关于更多html5标准化元素，可以参考：MDN标准化标签列表

  4. \的改进

    HTML内联框架元素 <iframe> 表示嵌套的浏览上下文，有效地将另一个HTML页面嵌入到当前页面中。

    H5新增sandbox、seamless、srcdoc属性，作者可以精确控制iframe元素的安全级

离线、存储

  1. Storage

    • Storage

      它是所有Storage实例（sessionStorage和globalStorage[location.hostname]）的构造函数

    • sessionStorage

      sessionStorage 是个全局对象，它维护着在页面会话(page session)期间有效的存储空间。只要浏览器开着，页面会话周期就会一直持续。当页面重新载入(reload)或者被恢复(restores)时，页面会话也是一直存在的。每在新标签或者新窗口中打开一个新页面，都会初始化一个新的会话。

      setItem和getItem存取数据

    • localStorage

      localStorage和sessionStorage的区别是存入的数据依然存在

    • cookie

      cookie其实不能算Storage的内容，不过放到这来做个区别。cookie一般由服务器生成，可设置实效时间。存放数据比较小，在4kb左右。每次都会携带在http头中。

  2. IndexedDB

IndexedDB 是一种低级API，用于客户端存储大量结构化数据(包括, 文件/ blobs)。该API使用索引来实现对该数据的高性能搜索。虽然 Web Storage 对于存储较少量的数据很有用，但对于存储更大量的结构化数据来说，这种方法不太有用。IndexedDB提供了一个解决方案。

具体可见：MDN:IndexedDB
