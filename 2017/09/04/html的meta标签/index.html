html的meta标签

发表于 2017-09-04 |

关于HTML的\标签

最近在准备校招的时候，对于移动端设备前端通常要加<meta name="viewport" content="width=device-width,initial-scale=1">

我发现对于这个meta标签我还是一无所知，所以在这篇博客中我把一些常用的meta相关知识整理在下面。

meta是什么？

metadata是元数据的意思，而meta标签代表了那些不能被HTML其他相关元素表示的数据信息。

The HTML element represents metadata that cannot be represented by other HTML meta-related elements, like\, \, \or\.

meta的用处

META标签用来描述一个HTML网页文档的属性，例如作者、日期和时间、网页描述、关键词、页面刷新等。它提供的信息虽然用户不可见，但却是文档的最基本的元数据。

简单的说，当浏览器或者搜索引擎爬虫找到这个网站的时候，可以凭借这个标签所包含的信息知道该如何访问这个网站。

meta的属性

meta的属性包括了name、http-equiv还有charset和content。

其中两个主要的属性是name和http-equiv, content是对前面两个属性的描述。而charset指定了网页所用的字符编码。

charset属性

  • 声明了页面的字符编码，这个属性的值必须在link之中。
  • 一般来说，建议使用UTF-8，<meta charset="UTF-8">
  • 声明字符编码的属性必须在head标签之中，并且在HTML的前1024个字节之内
  • 这个标签等同于html5之前的<meta http-equiv="Content-Type" content="text/html; charset=*IANAcharset*">，其中IANAcharset就是charset的属性。这种书写方式仍在使用，但在HTML5中更推荐charset的写法了

name属性

name属性主要用于描述网页的信息。一般使用方式和content结合

<meta name="参数" content="描述">

1、keywords（关键字）

定义了网页的关键字

2、application-name（应用名称）

定义了在网页上运行的应用的名称

3、author（作者）

4、description（关于网页的描述）

5、generator

定义了生成网页的软件标识

contains the identifier of the software that generated the page

6、robots

定义了爬虫的对页面的使用情况。

对于content的值可以取以下：

  • index：允许robot索引该页（默认）
  • noindex：不允许robot索引
  • follow：允许robot爬取该网页链接的网页（默认）
  • nofollow：和follow相反

更多的值可以参考mdn：meta

7、viewport

这个属性提示了视口的初始大小，并且只在移动设备使用。

关于这个属性的content的值，可以看下表

Value Possible subvalues Description
width 一个正整数或者是device-width 定义了视口的宽度，或者让视口适应设备的屏幕宽度。
height 一个正整数或者是device-height 定义了视口的高度。但是没有被任何浏览器采用（那还要它干嘛==）
initial-scale 一个在0.0 和10.0之间的正数 Defines the ratio between the device width (device-width in portrait mode or device-height in landscape mode) and the viewport size.定义了设备宽度和视口之间的比例。
maximum-scale 一个在0.0 和10.0之间的正数 Defines the maximum amount to zoom in. It must be greater or equal to the minimum-scale or the behavior is undefined. Browser settings can ignore this rule, and iOS10+ ignores it by default.最大缩放的比例。必须大于等于最小缩放的比例，IOS10+直接忽略了这个属性
minimum-scale 一个在0.0 和10.0之间的正数 Defines the minimum zoom level. It must be smaller or equal to the maximum-scale or the behavior is undefined. Browser settings can ignore this rule, and iOS10+ ignores it by default.定义了最小缩放比例。类似maximum-scale
user-scalable yes 或 no If set to no, the user is not able to zoom in the webpage. The default is yes. Browser settings can ignore this rule, and iOS10+ ignores it by default.定义了用户是否可以缩放屏幕。

http-equiv属性

这个属性可以包括HTTP header的相关名称，它可以通过描述改变服务端和客户端的一些行为。

1、cotent-language

建议使用html标签的lang属性<html lang="en">，而不使用该属性

2、content-type

建议使用前文的charset属性

3、refresh

在n秒后指向其他页面，使用方法：

<meta http-equiv="refresh" content="3;url=http://www.baidu.com/">

3秒后跳转到百度

4、set-cookie

Do not use this instruction, as it is obsolete. Use the HTTP header Set-Cookie instead.

不要使用这个，它已经被废弃了。使用HTTP头Set-Cookie代替

还有一些关于http-equiv的属性值，但是因为都可以在http头设置，这里便不再赘述。
