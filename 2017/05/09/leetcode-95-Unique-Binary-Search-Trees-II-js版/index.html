[leetcode 95]Unique Binary Search Trees II(js版)

发表于 2017-05-09 | 分类于 算法 |

这个系列是记录笔者刷leetcode的过程。作为一个前端，还是要会一点算法的。虽然之前就有在leetcode上刷题，但是都没记录下来。所以从现在开始就把刷过的题有些有意思的记录下来。一个是加深印象，另一个是等以后水平提高了可以看看自己当时写的是不是很丑陋什么的，哈哈。

今天带来的是，leetcode 95。题目意思是给一个范围n，求出从1到n的所有二叉搜索树。具体英文题目可以看上面链接。

笔者一开始的思路是，首先，有n个数，肯定要构建出一个[1…n]的数组，然后根据这个数组去构造出所有的二叉搜索树。其实给一组数，构建一个二叉搜索树很简单，当该组数的顺序不同，构建出的二叉搜索树也就不相同了。笔者想能不能把1到n的所有数进行全排列，然后构建。当然这样有个问题就是全排列产生的组合是大于该组数所形成的所有二叉搜索树的，所以我们要找到那些排列中形成二叉搜索树相同的组合并去重。

然后笔者在这个思路逗留了好久，发现想不出如何去重的方法。决定换个思路。

所谓的二叉搜索树，就是右子树是大于根节点的元素，左子树是小于根节点的元素。而左子树和右子树又分别是二叉搜索树，这就有递归的思想在里面。而要找到一组数所有的二叉搜索树，就是要选取该组数中不同的数当作根节点。这两个思路和在一起就有解决办法了。没看懂？没关系，先上代码：

首先，主体函数中的var arr = Array.from(Array(n).keys()).map(x=>++x);

会产生一个[1,2…n]的数组。这个数组是有序的，这对我们非常重要。接下来，我们写了一个函数generateT，将数组传入。这样，我们循环arr，选取不同的值作为根节点，而右子树为选取节点右边的值，左子树为选取节点左边的值。比如n=4，选2为根节点，则1为左子树的集合，3，4为右子树集合。接下来将1递归传入generateT，[2,3]递归传入generateT。

注意，因为要获得所有的搜索树，所以generateT所返回的应该是传入数组所产生的所有二叉搜索树的根节点的集合。这样，上一层函数就可以通过集合去生成新的根节点，根节点的左子树为递归返回的左子树，右子树同理。然后解决好，如果左右子树不存在的边界问题就好了。

因为当n为0时，generateT返回undefined。所以在处理下n为0的情况即可ac了。
