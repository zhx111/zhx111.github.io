模式4:Mediator

  • 将命令传递给所有订阅者
  • 与所有订阅者之间维持依赖关系
  • 从而让订阅者彼此之间相互独立

模式5:Composite

  • 对已有组件／库／功能的组合使用
  • 提升代码复用性的最常用手段

模式6:Partial Application

  • Function.prototype.bind
  • 部分应用／固化参数

Logging time

模式7:Decorate

  • 在原有API基础上增加额外的功能
  • 不改变底层库的基础上扩展功能的一种模式

让函数只被调用一次

debounce

避免一个函数被频繁的触发

模式8 Proxy

  • 保持原有API功能不变，改变执行过程和结果
  • 不改变底层库的基础上获得不同执行结果的手段

拦截异步数据

模式9:Aspect

  • 对代码执行过程进行监测和修改
  • 不改变代码主体执行流程的基础上控制执行过程

系统／环境默认的一些可控制点：

preventDefault()

Symbols

…

创建切面修改Swipable

preformance.watch 监控性能

总结Decorate、Proxy和Aspect

三者的联系：

  • Decorate强调的是声明和定义
  • Proxy强调是功能
  • Aspect强调的是运行过程

动态语言可以用Decorate来时实现Proxy和Aspect。

模式10:Lazy load

  • 一些加载耗时的资源进行分段、异步加载
  • 前端提升性能，减少响应用户的时间
  • 某些情况下减少网页流量的带宽和消耗
