什么是this

this不是编写时绑定，而是运行时绑定。它依赖于函数调用的上下文条件。

this的指向和函数的调用方式有关，当函数被调用的时候，一个被称为execution context（执行上下文）的活动记录被建立，这个记录包含函数是从何处（调用栈 —— call-stack）被调用的，函数是 如何 被调用的，被传递了什么参数等信息。这个记录的属性之一，就是在函数执行期间将被使用的 this 引用。

Web缓存机制

前言：

最近在看HTTP协议的相关知识，其中HTTP首部字段中有Cache-Control、If-None-Match、ETag等字段。这些都和Web缓存有关。之前博主一直对缓存这块知识有点模糊，所以这篇博客就试着理清并总结一下Web缓存的相关知识，如果有些地方写得不对的地方希望能指出。

为什么需要缓存

虽然现在网速在不断提升，但是网页的性能还是十分重要。如果在一个网页的加载中，能对于不变的静态资源进行缓存，在之后的加载中就可以省去下载的过程，直接读取缓存，减少流量消耗。

当然，缓存也会带来问题。如果资源更新了，但是依旧从缓存读，就会造成资源更新不及时的问题。一般是可以修改资源的文件名（比如通过webpack在文件名中加一个hash），或者控制资源过期时间。

第1章 了解Web及网络基础

1.4 与HTTP关系密切的协议：IP、TCP和DNS

IP协议作用是把各种数据包传送给对方。而要确保传送到对方那里，则需要满足各类条件。其中最重要的条件是IP地址和MAC地址。

ARP协议：一种用于解析地址的协议，根据通信方的IP地址就可以反查出对应的MAC地址。

OSI模型中ARP协议属于链路层；而在TCP/IP模型中，ARP协议属于网络层

确保可靠性的TCP协议

TCP提供字节流服务。

TCP的建立连接三次握手：

1、发送端首先发送一个带SYN标志的数据包给对方。

2、接受端收到后，回传一个带有SYN／ACK标志的数据包以示传达确认信息。

3、最后，发送端再回传一个带ACK标志的数据包，代表“握手”结束。

如果在握手过程中某个阶段莫名中断，TCP协议会再次以相同的顺序发送相同的数据包。

插入排序

平均时间复杂度为O(N^2)，但是在基本预先排序，那么插入排序会很快。

希尔排序

希尔排序使用一个增量序列，在使用增量为h的一趟排序之后，对于每一个i，我们都有a[i]<=a[i+h]。所有相隔元素h的元素都被排序，此时称文件是h排序的。

h为一个增量序列，如1、3、5。当h为1时即完成排序。

希尔本人建议的序列是最大的h为N/2，hk=hk+1/2

CSS 2.1（CSS权威指南）

第二章选择器

  • 通配选择器为星号（*）
  • 元素，类，ID选择器
  • 属性选择器
    • 简单属性选择器 h1[class]{color:red;}
    • 根据具体属性值选择 p[class="urgent warning"]{}不同于类选择器，是一个完全串的匹配
    • 根据部分属性值选择
      • [foo~="bar"] 选择foo属性值包含“bar”的所有元素
      • [foo^="bar"] 选择foo属性值以“bar”开头的所有元素
      • [foo$="bar"] 以bar结尾
      • [foo*="bar"] 包含子串“bar”
    • 特定属性选择类型
  • 后代选择器（加一个空格）选择继承的所有元素
  • 选择子元素（使用大于符号>）只是子元素，不包括继承的
  • 选择相邻兄弟元素 （使用+号）只能选择两个相邻兄弟中的第二个元素
  • 伪类和伪元素
    • 链接伪类
      • :link有href属性的超链接
      • :visited作为以访问链接的所有锚
    • 动态伪类
      • :focus当前拥有输入焦点的元素
      • :hover
      • :active被用户输入激活的元素
    • 伪类顺序 l-v-f-h-a
    • 选择第一个子元素
      • :first-child 常见的错误时认为例如p:first-child之类的选择器会选择p元素的第一个子元素。但实际是作为第一个子元素的p元素
    • 伪元素选择器
      • 设置首字母样式:first-letter
      • 设置第一行样式:first-line
      • 设置之前和之后元素的样式:after :before
