用Hexo+Github pages搭建自己的博客

发表于 2017-01-08

前言

一直很想建一个属于自己的个人博客，在经过一番筛选之后，我选择了Hexo+Github Pages这个组合来搭建我的博客。选用这个的好处是因为Github是每个程序员必备的，而Github提供了一个可供展示自己静态网站的稳定空间，我们可以利用这个建立自己博客。而Hexo正如它的首页所说，是一个快速、简洁而且高效的博客框架。

建站过程

下面介绍了本博客的建站过程，想得到更多详细的信息，请查看hexo的官网：https://hexo.io/docs/index.html

另外本人使用系统为macOS

建站环境配置

安装Hexo的过程非常简单，你只需要在电脑中拥有：

  • Node.js
  • Git

Node.js是一个基于Chrome V8引擎的javascript运行环境，其提供的npm是目前最大的开源包管理系统；Git则是一个现在非常流行分布式项目管理版本控制系统，我们熟知的Github就是基于git的最大的开源项目托管平台（这两个应该都非常熟悉了）

开博感言

发表于 2017-01-08

写在前面：

这是我博客的第一篇文章，并不想多写什么，只想谈谈自己的感悟，所以这篇文章没有什么干货。

​ 这是我第一次搭建起属于自己的博客，当我有了这样一个平台，可以记录自己平时遇到的各种问题，可以写下学习过程中的所思所想，甚至是可以不着边际地畅所欲言，自然会有一种莫名的兴奋。

​ 但是兴奋过后，我也开始有一点慌张。好久没有写过文章的我会不会因文笔不佳被人嘲笑，刚刚跨专业进入技术行业的我会不会因为记录和分享的博文过于简单而被人看轻（尤其是技术圈往往有一种鄙视链的存在，例如XXX是最好的语言，哈哈），但是这个博客既然开了就不能荒废，而第一篇文章，就讲讲关于我自己、关于开这个博客目的，还有关于未来的规划，各位看官见笑了。

关于我

​ 让一个人在三言两语中总结清楚他自己还是挺难的一件事，乐观的人常常高估自己，悲观的人则往往会贬低自己。而我呢，总的来说是一个悲观中带着点乐观的人。大学录取被退档，去了一个马马虎虎的学校和学着一个不太喜欢的专业，这段经历还是给我打击挺大的，也迷茫了蛮久。终于在大三的时候决定跨专业考研，经过几个月的努力，来到了南京大学的软件工程专业。所以现在的我是一名软件工程的研究生，勉勉强强半只脚跨进了这个行业。

​ 现在的我，只能说大概了解计算机专业的基础知识，对一两门编程语言慢慢开始熟悉，我知道软件这个行业的知识真的是星辰大海般深远，还是需要慢慢去学习。既然是技术博客，别的方面就不多说了，目前感兴趣的方向是前端，我认为为用户设计出极具艺术的交互界面是一件非常有成就感的事。希望往后的几个月能逐渐成长起来。

开博目的

​ 关于为什么要写博客，大家可以看看阮一峰老师的这篇文章为什么要写博客？。下面谈谈我自己的理解。

  • 写博客能够记录下你学习的过程，将新学的一个知识写下来的过程会让你更好的掌握它，也有助于形成你自己的体系结构；
  • 写博客可以提升你表达观点的能力，在现在社会中，能清楚表达自己观点的人是非常可贵的；
  • 最后，写博客是一件非常酷的事。在浩瀚的互联网上有那么一个角落，记录了你的生活，你的印记，实在是一件鹅妹子嘤的事啊。

未来规划

​ 这个博客将来的内容我想主要包括这几个类型：

  • 一是学习某个语言或者技术的总结积累（包括外文文章翻译）
    • javascript
    • css
    • react
    • ……
  • 二是构建一个项目的过程
    • 博客
    • ……
  • 三是一些读书笔记
  • 四是一些生活随感、杂记

​ 希望写博客这件事能够坚持下来，先给自己定一个小目标，每周两篇博客，以后也要努力了。
